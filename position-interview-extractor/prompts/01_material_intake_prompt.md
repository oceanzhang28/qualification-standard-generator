# 材料接收提示词

## 目标

确认是否已收到可开始提炼的 HR 岗位基本信息表和完整访谈转写稿；只收集阻断提炼的缺失信息。

## 执行步骤

1. 检查用户是否已提供 [HR 岗位基本信息表](../templates/hr_basic_info_template.md)。表单未提供时，完整展示该表单；表单已部分填写时，只列出缺失的阻断字段，不重复展示完整表单或要求重填已提供内容。当前实际覆盖职级、受访专家姓名/角色、受访专家主要熟悉职级和访谈日期等可选缺口可单独标明，但不得要求用户补充。
2. 先检查岗位性质：只接受“专业类岗位”或“管理岗位”。确认值为“管理岗位”时，简要说明本 Skill 仅处理专业类岗位，拒绝继续处理，并且不再检查或追问其他材料、不开始事实提炼、不输出岗位事实审核稿或清洁岗位信息包。确认值缺失、含糊或不在枚举内时，状态为“材料待补充”，只要求用户明确选择“专业类岗位”或“管理岗位”。只有确认值为“专业类岗位”时才继续。不得根据岗位名称、职类、职务子类、所在部门或访谈内容推断岗位性质。
3. 岗位性质通过后，检查是否已收到一份完整访谈转写稿。转写稿可以直接粘贴，或提供可读取的文本或 Markdown 文件；必须包含访谈全文，不接受摘要、提纲或片段代替全文。
4. 仅追问阻断提炼的信息：职类、职务子类、所在部门、岗位名称、岗位性质和完整访谈转写稿。不要为了补全可选信息而追加问题。
5. 当前实际覆盖职级、受访专家姓名/角色、受访专家主要熟悉职级和访谈日期可标明为可选信息缺口，但不作为开始提炼的前置条件，也不主动追问。
6. 明确说明：职级差异信息可以只覆盖部分职级、部分任务或完全缺失；这不会阻断事实提炼，未覆盖部分会标记为 `[缺失]` 或待确认，不能推断为不适用。
7. 检查职级表述：可直接作为适用职级的值只有专员、主管、经理、总监。遇到 P5、M2 或数字等非标准内部职级时，保留原值并标记“待映射”；映射不阻断事实提炼，但必须在进入“已确认”前由审核人明确确认。不得从受访专家主要熟悉职级推断映射。

## 状态与输出

- 缺少表单、表单必填字段或完整转写稿，或岗位性质含糊、超出枚举时，状态为“材料待补充”。列出缺失的阻断项，拒绝开始提炼、拒绝输出岗位事实审核稿或清洁岗位信息包。
- 岗位性质明确为“管理岗位”时，输出简洁的范围说明并停止，不把该输入转入“材料待补充”或后续审核状态。
- 只有岗位性质明确为“专业类岗位”，且其余表单必填字段和完整转写稿都齐全时，才说明材料可进入事实提炼，并交由下一提示词处理。
- 表单与转写稿出现岗位身份或组织设置冲突时，保留双方原始表述，作为后续“待人工审核”事项；不要自行选择一方。
