# 岗位事实提炼提示词

## 前置条件

仅在 HR 岗位基本信息表和完整访谈转写稿均已齐全，且岗位性质已确认且为“专业类岗位”时执行。岗位性质为“管理岗位”或尚未明确时不得执行。读取 `rules/key_task_extraction_rules.md`、`rules/level_difference_extraction_rules.md`、`rules/validation_rules.md` 和 `templates/fact_review_template.md`。

## 固定执行顺序

```text
识别基本信息
→ 分段标记转写主题
→ 提取候选任务及证据
→ 合并或拆分任务
→ 排除非核心或越界事项
→ 形成 5–10 条任务
→ 提炼三层稀疏职级差异
→ 校验
→ 输出岗位事实审核稿
```

## 执行要求

1. 识别并保留表单中的岗位身份、岗位性质和组织信息；岗位性质固定写为已确认的“专业类岗位”。其他信息与转写稿不一致时保留双方原始表述并标记 `[存在冲突]`，不得自行选择。职级值不是专员、主管、经理或总监时，保留在“待映射的内部职级”并将映射确认状态设为“待确认”；不得从专家熟悉职级推断映射。
2. 按岗位定位、工作对象、工作流程、产出、关键任务、边界、复杂场景和职级差异分段标记转写主题。
3. 为每个候选任务提取场景/输入、动作、产出、评价要求、主责与协作边界、复杂性、来源标签和证据摘要。证据摘要必须简洁概括可定位的事实，不要长篇引用转写原文。
4. 按任务目的、产出和主责边界合并同一端到端工作的连续步骤；目的、产出或主责边界实质不同才拆分。排除通用能力、任职条件、人员管理要求、临时杂务和协作方主责事项。
5. 最终岗位关键任务必须为 5–10 项。不能满足时按规则保留候选任务/证据并输出“需要补充材料”，不得用推测或通用能力凑数。
6. 只从材料提炼“岗位整体”“工作领域或任务组”“单项任务”三个层级的职级差异；每条记录绑定最小充分作用范围。职级差异可稀疏、部分覆盖或完全缺失，未覆盖信息填写 `[缺失]`，不得使用通用规则补齐或写成不适用。逐项列出未覆盖的职级、任务组或任务：只要存在一项，将“允许生成 Skill 使用内置通用职级规则补充”设为“待确认”；没有任何未覆盖项时设为“不需要”。事实提炼阶段不得自行填写“是”或“否”。
7. 对每项事实使用 `[专家明确]`、`[AI归纳]`、`[缺失]` 或 `[存在冲突]`；`[AI归纳]`必须能追溯到具体材料。不得生成或混入系统预测的职级差异。
8. 依照校验规则完成校验后，使用事实审核模板输出 `# 岗位访谈提炼审核稿`，审核状态固定为“待人工审核”。输出前必须按实际任务条目复核审核稿中每处任务数量声明，并逐项扫描所有 `[存在冲突]` 标记，确认信息缺口、冲突摘要和人工确认事项覆盖每个冲突任务与字段；发现不一致时先修正并重新执行全部校验，不得呈现未通过一致性校验的审核稿。保留来源、证据摘要、信息缺口、冲突和建议人工确认事项；不得直接输出清洁岗位信息包。
