# 清洁岗位信息包格式化提示词

## 前置检查

先检查审核稿的 `审核状态：已确认`。未经明确确认，或状态不是“已确认”时，拒绝格式化清洁岗位信息包，并继续停留在“待人工审核”。

## 格式化要求

1. 使用 `templates/clean_handoff_template.md` 输出清洁岗位信息包。
2. 只保留已确认的岗位事实；省略转写证据、原文引述、来源标签、AI 推理过程和审核过程描述。
3. 将 5–10 项已确认任务映射到现有生成 Skill 的“岗位关键任务”；将主要工作对象、主要工作流程、主要产出物映射到“岗位特殊说明”；将协作方、典型复杂场景和关键业务指标映射到可选信息；将分层职级差异映射到“特殊职级差异”。
4. 以岗位整体、工作领域或任务组、单项任务三个作用范围保留稀疏职级差异；不得把一个范围的事实扩展到其他任务或职级。
5. 若全部职级差异均无可靠信息，在“职级差异信息说明”中明确写明“未获得可靠的职级差异信息；允许生成 Skill 使用内置通用职级规则补充：是 / 否”，不得留空或虚构差异。
6. 已明确标注为不阻断生成但尚未解决的问题，单列在“其他可选信息”的“待人工复核事项”中；不得伪装成已确认事实。
7. 输出前确认关键任务仍为 5–10 项，岗位基本信息完整，且不存在校验规则定义的阻断性未解决冲突。
