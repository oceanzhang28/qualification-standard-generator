# 输入材料要求

开始提炼前，必须同时提供以下材料：

1. 已填写的 [HR 岗位基本信息表](../templates/hr_basic_info_template.md)。
2. 一份完整访谈转写稿。

转写稿可直接粘贴为文本，也可提供可读取的文本或 Markdown 文件。v1 不进行音频转写或视频转写。

## 必填信息与缺失处理

下列任一信息缺失时，状态为“材料待补充”，不得开始提炼：

- 职类
- 职务子类
- 所在部门
- 岗位名称
- 岗位性质
- 完整访谈转写稿

“当前实际覆盖职级”“受访专家姓名/角色”“受访专家主要熟悉职级”和“访谈日期”建议提供；其中职级差异信息缺失或仅覆盖部分职级时，不阻断提炼。未被证据覆盖的职级必须标记为缺失或待确认，不得推断为不适用。

## 岗位性质口径

- 岗位性质只接受“专业类岗位”或“管理岗位”。只有明确填写“专业类岗位”才可进入事实提炼。
- 明确填写“管理岗位”时，简要说明本 Skill 仅处理专业类岗位并停止处理，不输出岗位事实审核稿或清洁岗位信息包。
- 岗位性质缺失、含糊或不在枚举内时，状态为“材料待补充”，要求用户明确选择“专业类岗位”或“管理岗位”。
- 不得根据岗位名称、职类、职务子类、所在部门或访谈内容推断岗位性质。

## 适用职级口径

- 可交接的适用职级只能是“专员”“主管”“经理”“总监”，可多选，不得加入其他值。
- 表单若使用 P5、M2 或数字等非标准内部职级，不阻断事实提炼；将原值保留在“待映射的内部职级”，将映射关系列为审核项。
- 审核人必须明确确认每个内部职级与固定四级的映射，才能进入“已确认”。不得根据受访专家主要熟悉职级推断映射或适用范围。

## 信息冲突

岗位基本信息表用于识别岗位身份和组织设置，转写稿用于提供岗位事实证据。两者信息不一致时，保留双方原始表述，并作为待人工确认事项；不得自行选择或覆盖任一来源。
