# 审核工作流

工作流使用以下状态：`材料待补充`、`待人工审核`、`已确认`。其中“材料完整”是材料检查通过的条件，不是可跳过人工审核的确认状态。

岗位性质是进入工作流的前置门禁：只有用户明确填写或确认“专业类岗位”才可进入事实提炼；缺失、含糊或非枚举值时保持“材料待补充”并要求明确选择，不得根据岗位名称或其他材料推断。明确为“管理岗位”时，简要说明本 Skill 仅处理专业类岗位并停止，不进入事实提炼、人工审核或清洁交接。

```text
材料待补充
  → 材料完整
  → 待人工审核
  → 用户提出修改：继续停留在待人工审核
  → 用户明确确认：已确认
  → 输出清洁岗位信息包
```

## 状态处理

- 材料缺少必填项时，保持“材料待补充”，仅说明需补充的内容。
- 岗位性质明确为“专业类岗位”且材料完整后，输出岗位事实审核稿并进入“待人工审核”。
- 用户提出修改时，按修改意见更新审核稿，状态继续保持“待人工审核”。
- 仅当用户作出明确的岗位事实确认，且下列独立确认项已按当前审核稿解决后，状态才变为“已确认”，随后输出清洁岗位信息包。

## 独立确认项

- **规则补充授权**：当前审核稿存在未覆盖职级、任务组或任务时，必须记录用户明确选择的“是”才能交接。通用岗位事实确认不代表授权。选择“否”时保持“待人工审核”，说明需先补充并确认这些缺失事实。当前审核稿没有任何未覆盖项时，记录“不需要”，不代为选择“是”或“否”。
- **非标准职级映射**：材料包含 P5、M2 或数字等固定四级之外的职级时，必须记录原值、映射关系，并取得审核人对具体映射的明确确认。通用确认不能替代映射确认，也不得从专家熟悉职级推断映射。

## 确认口径

可接受的岗位事实确认包括：“确认岗位信息”“审核通过”“按此版本提交”，或意思无歧义的等同表述。这些表述只确认事实稿，不自动填写规则补充授权或非标准职级映射。

“可以看看下一步”“先这样”“差不多”等含糊回复不构成确认；此时继续保持“待人工审核”，并请用户明确确认或提出修改。
