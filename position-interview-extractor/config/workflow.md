# 审核工作流

工作流使用以下状态：`材料待补充`、`待人工审核`、`已确认`。其中“材料完整”是材料检查通过的条件，不是可跳过人工审核的确认状态。

```text
材料待补充
  → 材料完整
  → 待人工审核
  → 用户提出修改：继续停留在待人工审核
  → 用户明确确认：已确认
  → 输出清洁岗位信息包
```

## 状态处理

- 材料缺少必填项时，保持“材料待补充”，仅说明需补充的内容。
- 材料完整后，输出岗位事实审核稿并进入“待人工审核”。
- 用户提出修改时，按修改意见更新审核稿，状态继续保持“待人工审核”。
- 仅当用户作出明确确认后，状态才变为“已确认”，随后输出清洁岗位信息包。

## 确认口径

可接受的明确确认包括：“确认岗位信息”“审核通过”“按此版本提交”，或意思无歧义的等同表述。

“可以看看下一步”“先这样”“差不多”等含糊回复不构成确认；此时继续保持“待人工审核”，并请用户明确确认或提出修改。
