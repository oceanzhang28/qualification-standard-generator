# 关键任务提炼规则

## 目标与事实标签

从岗位基本信息表和完整访谈转写稿中提炼岗位实际承担的 5–10 项关键任务。关键任务是岗位持续承担、能够体现专业价值和主责边界的完整工作；不是任职条件、通用能力或协作方职责。

事实提炼阶段统一使用下列来源标签：

| 标签 | 使用条件 |
| --- | --- |
| `[专家明确]` | 表单或转写稿中有可定位的直接表述。 |
| `[AI归纳]` | 对同一材料中多处具体事实做合并、标准化或概括；不得新增原材料未支持的任务、边界、产出或职级事实。 |
| `[缺失]` | 材料没有足够证据，无法确认某项任务或其必要子字段。 |
| `[存在冲突]` | 表单与转写稿，或材料中的不同表述相互矛盾且尚未人工确认。保留各方原话，不自行取舍。 |

## 候选任务识别

1. 从岗位价值、典型工作周期、反复出现的工作内容、具体项目或问题案例、主要产出物和主责边界中提取候选任务。
2. 优先保留在不同表述或案例中反复出现的工作，以及被明确说明为岗位核心、常态或必须承担的工作。
3. 即使只出现一次，低频但对业务结果、专业风险、关键决策或机制建设有重大影响的专业工作也应保留，并注明其触发场景。
4. 个别案例只有在材料能够证明其代表岗位常态时才可作为任务事实；否则仅可作为理解上下文的证据，不单独扩写为常态任务。

## 合并与拆分

- 当候选事项的目的、工作对象、主要产出和主责边界实质相同，仅是同一完整任务中的先后步骤或不同说法时，应合并为一项任务。
- 描述端到端工作时，保留一个完整任务，不把调研、分析、沟通、执行、复盘等连续步骤机械拆成多个关键任务。
- 当目的、产出或主责边界之一发生实质差异时，应拆分为不同任务；不同触发场景导致独立产出或独立责任的，也应拆分。
- 合并或拆分后，检查每项任务的输入、动作、产出和主责边界均可从材料追溯，且任务之间没有重复归属。

## 排除项与边界

不得列为关键任务：

- 临时跑腿、偶发行政事项或与岗位专业价值无关的杂务；
- 沟通、抗压、协作、学习等通用能力或个人特质；
- 带人、绩效管理、团队管理等人员管理要求；
- 由协作方主责，本岗位仅提供输入、配合、审批支持或接收结果的工作；
- 学历、年限、性格、领导力、价值观等任职条件。

若材料无法判断本岗位是主责还是协作，任务边界标记为 `[缺失]`；若不同材料说法不一致，标记为 `[存在冲突]`，均不得把协作方职责写成本岗位主责。

## 输出要求

输出 5–10 项关键任务。每项至少包含：

```text
任务名称：
标准化描述：
主责边界：
信息来源：
证据摘要：
待确认或缺失：
```

- 任务名称使用简洁的业务动作和对象命名，避免使用能力词。
- 标准化描述按以下结构组织：`基于【输入/依据/场景】，完成【关键动作】，形成【产出/结果】，确保或支撑【评价要求/业务目的】`。
- 如输入、依据、场景、产出、评价要求或业务目的缺少证据，对应子字段保留 `[缺失]` 并写入“待确认或缺失”；不得为使句子完整而补造事实。
- 信息来源使用本文件定义的四种事实标签之一。`[AI归纳]` 仍须在证据摘要中列出被归纳的具体材料表述。

## 数量不足或过多

- 最终“岗位关键任务”清单必须为 5–10 项，不得输出少于 5 项或多于 10 项的最终清单。
- 少于 5 项时，先检查是否错误合并了目的、产出或边界不同的任务；仍不足时，将现有候选事项和证据单独保留在“候选任务/证据”区，标记 `[缺失]` 并要求补充材料，不以通用能力或推测任务凑数。
- 多于 10 项时，先按本规则合并同一完整任务的连续步骤和同类表述；不得删除已证实的低频高影响专业工作。若仍无法形成 5–10 项最终清单，将完整候选清单及证据单独保留在“候选任务/证据”区，要求补充材料或人工明确任务分组，不得将超出数量的候选事项作为最终清单输出。
- 最终清单未达到 5–10 项时，校验结论为“需要补充材料”，人工确认和清洁岗位信息包交接均保持阻塞。
