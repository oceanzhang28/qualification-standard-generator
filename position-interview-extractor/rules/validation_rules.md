# 岗位事实稿校验规则

## 校验前提

校验对象为“待人工审核”的岗位事实审核稿。校验只核验材料支持度、任务边界和字段完整性，不把通用任职资格框架写入岗位事实。事实阶段来源标签含义如下：

| 标签 | 使用条件 |
| --- | --- |
| `[专家明确]` | 表单或转写稿中有可定位的直接表述。 |
| `[AI归纳]` | 对同一材料中多处具体事实做合并、标准化或概括；不得新增原材料未支持的任务、边界、产出或职级事实。 |
| `[缺失]` | 材料没有足够证据，无法确认某项任务或其必要子字段。 |
| `[存在冲突]` | 表单与转写稿，或材料中的不同表述相互矛盾且尚未人工确认。保留各方原话，不自行取舍。 |

## 检查项

| 检查项 | 通过条件 | 不通过时处理 |
| --- | --- | --- |
| 任务数量 | 最终“岗位关键任务”清单为 5–10 项；少于 5 或多于 10 项时已按提炼规则复核合并、拆分和低频高影响任务。 | 不得输出数量不合规的最终清单。将候选事项及证据单独保留，校验结论为“需要补充材料”；人工确认和清洁岗位信息包交接保持阻塞，直至最终清单为 5–10 项。 |
| 任务描述完整性 | 每项任务有任务名称、关键动作、工作对象，以及可追溯的产出或业务目的。 | 缺少字段标记 `[缺失]`，不得用推测补齐。 |
| 重复与边界重叠 | 任意两项任务不因连续步骤或同义说法重复；目的、产出或主责边界不同的工作未被错误合并。 | 合并或拆分后重新校验，并保留可追溯证据。 |
| 主流程与产出覆盖 | 任务清单覆盖岗位主要工作流程、核心产出和被证实的低频高影响专业工作。 | 记录遗漏点并要求补充材料或人工确认。 |
| 协作方职责泄漏 | 每项任务说明本岗位主责边界；协作方主责的决策、交付或执行未被写成本岗位任务。 | 主责已明确属于协作方时，删除该活动，或仅保留本岗位有证据支持的贡献后重写任务；仅在边界确实无法确认时标记 `[缺失]`，仅在材料相互矛盾时标记 `[存在冲突]`。 |
| 禁止的通用能力内容 | 不含沟通、抗压、协作、学习等通用能力，不含学历、年限、性格、领导力、价值观或人员管理要求。 | 删除或移出关键任务，必要时要求补充实际专业工作。 |
| 事实支持度 | 每项任务与职级差异均有 `[专家明确]`、可追溯的 `[AI归纳]`、`[缺失]` 或 `[存在冲突]` 标签和证据摘要。 | 未被材料支持的内容删除或标记 `[缺失]`；不得创建 `[规则预测]`。 |
| 职级差异层级绑定 | 每条差异均绑定“岗位整体”“工作领域或任务组”或“单项任务”，并在适用关键任务字段中明确范围。 | 补充绑定；无证据时标记 `[缺失]`。 |
| 缺失与不适用 | 未覆盖的职级、任务或层级标记 `[缺失]`；`[不适用]`均有岗位实际范围或专家明确证据。 | 无证据的 `[不适用]`改为 `[缺失]`，并要求补充或人工确认。 |
| 表单与转写稿冲突 | 岗位身份、组织设置、任务边界和职级范围的矛盾均保留双方原始表述并标记 `[存在冲突]`。 | 不得自行选择任一来源；状态保持“待人工审核”。 |
| 交接兼容性 | 保留已接受输入中的职类、职务子类、所在部门、岗位名称、当前实际覆盖职级、受访专家姓名/角色、受访专家主要熟悉职级和访谈日期；同时保留关键任务、来源、证据摘要、缺失/冲突和人工确认状态。 | 缺失必填岗位身份或完整转写稿时转为“材料待补充”；其余字段按事实状态标记并在审核稿中保留。 |

## 校验结论

校验结果只能使用以下之一：

- `可以进入人工审核`：材料必填项齐全，最终“岗位关键任务”清单为 5–10 项，任务和差异均可追溯；可保留 `[缺失]` 或 `[存在冲突]`，但必须明确列为人工审核事项。
- `需要补充材料`：缺少输入必填项，最终“岗位关键任务”清单少于 5 项或多于 10 项，或关键任务的主流程、产出、主责边界等缺口无法在现有材料中确认。此结论下不得人工确认或输出清洁岗位信息包。
- `存在重大冲突`：岗位身份、主责边界、职级实际覆盖范围或关键产出存在未解决的实质矛盾，不能形成可审核的统一事实稿。

无论校验结论为何，未经用户明确确认，人工确认状态不得写为“已确认”，也不得输出清洁岗位信息包。
