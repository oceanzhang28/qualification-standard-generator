# 双 Skill 协作使用指南

本项目用两个独立 Skill 完成“访谈事实提炼”和“任职资格生成”。两个阶段之间设置人工审核关口，未经确认的访谈提炼结果不能直接进入生成阶段。

## 适用范围

- 适用于公司专业类岗位，不用于管理岗位任职资格体系。
- 固定职级为专员、主管、经理、总监。
- 职级差异不是访谈必填项；专家只能说明部分职级或整体差异时，应如实保留，不强制补齐到每项关键任务。

## 第一步：提炼并审核岗位事实

使用 `position-interview-extractor`，至少提供：

1. 岗位基本信息，包括岗位名称、所属部门、岗位性质等。
2. 一位岗位专家的完整访谈转写稿；建议按照 90 分钟访谈提纲开展访谈。
3. 已知的职级差异信息，可以位于岗位整体、任务组或单项关键任务层级。

Skill 先检查材料，再提炼岗位定位、5–10 项候选关键任务、专家明确表达的职级差异，以及信息缺口和冲突，输出“岗位事实审核稿”。HR 或业务人员必须检查：

- 关键任务是否覆盖岗位主要价值，并准确描述触发场景、输入、动作、产出和评价标准。
- AI 是否把推断误写成专家事实。
- 职级差异的适用范围和信息来源是否正确。
- 是否存在需要修订的冲突或缺口。

只有用户明确确认审核稿，并对缺失职级信息是否允许生成 Skill 按通用规则补充作出选择后，才能输出“清洁岗位信息包”。不同意系统补充时，不得继续交接。

## 第二步：生成任职资格标准

将第一步确认后的完整“清洁岗位信息包”提交给 `qualification-standard-generator`。生成 Skill 应按以下优先级使用职级信息：

1. 专家明确说明且经人工确认的岗位事实。
2. 经确认的任务组或岗位整体差异。
3. 仅在用户同意补充时，使用系统内置的通用职级差异规则预测缺口，并保持可审核性。

生成分为两个阶段：

1. 阶段一输出行为模块与行为要项框架，人工确认后才能继续；最多支持 2 轮调整。
2. 阶段二输出专员、主管、经理、总监四级行为标准，并提供 Markdown 预览、TSV 和校验结果。

## 关键交接原则

- 不要求每个关键任务都有四职级差异，也不把未访谈到的内容伪装成专家结论。
- 审核稿用于人与 AI 校正事实；清洁岗位信息包才是生成 Skill 的正式输入。
- 两个 Skill 通过 Markdown 信息包协作，目前不自动相互调用。
- 输入信息变化会影响最终标准；岗位事实审核不能省略，也不能由 AI 自行代替人工确认。

详细访谈问题见 [`../position-interview-extractor/references/interview_guide.md`](../position-interview-extractor/references/interview_guide.md)，运行规则分别以两个 Skill 的 `SKILL.md` 和其直接引用文件为准。
