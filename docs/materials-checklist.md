# qualification-standard-generator 材料准备清单

## 1. 准备原则

你不需要提前把所有内容整理成 Skill 文件格式。你只需要提供业务原始材料和必要判断，我来负责转换、拆分、清洗和放入 Skill。

材料优先级：
1. 先准备首批成熟或可参考岗位 Excel 案例。
2. 再补充每个案例的基本信息和质量判断。
3. 再补充公司内部对职级差异、行为标准写法的特殊要求。
4. 最后逐步补充更多岗位序列规则和案例。

已补充的统一规则：
- 公司制定任职资格的岗位均为专业类岗位，管理岗位不适用本任职资格体系。
- 用户输入和案例元数据使用“职类”“职务子类”，不再使用“职族”“职位类别”作为必填字段。
- 职位类别在 v0.1 中标记为“不适用”。
- 所有岗位固定包含“组织贡献”行为模块，并固定拆分为“平台/流程建设”“人员培养”2 个行为要项。
- 职位图谱文件路径：`/Users/oceanzhang/Documents/任职资格AI生成工具/任职资格参考标准/职位图谱.xlsx`。

## 5. Excel 模板要求

每个 Excel 文件对应一个岗位。

Excel 必须由你线下先整理为统一 A-G 列结构：

| 列 | 字段 |
|---|---|
| A | 序号 |
| B | 行为模块 |
| C | 行为要项 |
| D | 专员行为标准 |
| E | 主管行为标准 |
| F | 经理行为标准 |
| G | 总监行为标准 |

注意：
- 第一行建议使用上述字段名。
- 一个文件只放一个岗位。
- 不要求每个行为要项在四个职级下都有内容。
- 如果某个职级不参与该行为要项，对应单元格可以为空。
- 空白表示原始案例中该职级不承担该行为要项或暂无标准。
- 不需要为了模板完整而人工补写空白职级。

## 6. 案例内容处理规则

我转换案例时会遵守以下规则：

- 保留原始编号。
- 保留原始行为模块、行为要项和行为标准文字。
- 空白职级单元格保持为空，不自动补写。
- 不擅自改写成熟案例原文。
- 对于标记为“行为模块与要项成熟，行为标准不成熟”的案例，只重点学习模块拆解、要项拆解和职级差异，不把低颗粒度行为标准作为优质写法样例。
- 如果发现明显格式问题，会先列出问题让你确认。
- 如果 Excel 不是 A-G 统一结构，转换脚本应停止并提示字段不匹配。

## 7. 案例库转换后的产物

每个案例会被转换为四类内容。成熟案例可用于学习完整写法；不完全成熟案例只按质量标签学习其可迁移部分。

### 7.1 完整岗位案例

路径：

```text
references/case_library/full_positions/
```

用途：
- 学习一个岗位完整的模块、要项和职级标准结构。
- 参考整体颗粒度。
- 参考职级递进方式。

### 7.2 行为模块片段

路径：

```text
references/case_library/snippets/by_module/
```

用途：
- 学习岗位关键工作领域如何拆分。
- 为相似岗位提供模块命名和边界参考。

### 7.3 行为要项片段

路径：

```text
references/case_library/snippets/by_behavior_item/
```

用途：
- 学习模块下的具体工作项如何拆分。
- 参考行为要项粒度。

### 6.4 职级递进片段

路径：

```text
references/case_library/snippets/by_level_progression/
```

用途：
- 学习同一行为要项下，专员、主管、经理、总监之间如何拉开差异。
- 避免四个职级只是换词。
