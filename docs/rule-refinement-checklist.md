# 任职资格 Skill 规则修订清单

## 1. 本次测试暴露的问题

### 1.1 职级差异动作过于单一

问题表现：
- 经理级大量使用“统筹”开头。
- 同一职级在不同工作项下动作差异不足。
- 职级差异看起来像固定套词，而不是根据具体工作内容变化。

需要补充：
- 不同行为动作类型下，专员/主管/经理/总监分别可以使用哪些动作词。
- 同一职级不能总是使用同一个动作词。
- 经理级除了“统筹”之外，哪些场景应使用“主导、制定、判断、优化、解决、推动、把关、沉淀”等动作。

### 1.2 行为标准颗粒度过粗

问题表现：
- 每个行为要项下每个职级基本只写 1 条行为标准。
- 精简行为模块和行为要项后，原本被合并的工作细项没有在行为标准中用编号展开。
- 例如“产品设计”这个行为要项，可以拆成设计方案、图稿表达、样品协同、样品调整等细项，但生成时被压缩成一句。

需要补充：
- 行为要项被合并后，如何判断应拆成几条行为标准。
- 哪些行为要项必须按工作细项编号展开。
- 每个单元格内 1、2、3 条标准分别对应什么拆分逻辑。

### 1.3 行为模块精简后，细项不能丢失

问题表现：
- 阶段一框架精简后，阶段二生成也跟着变粗。
- 模块/要项精简是结构层级上的合并，不代表内容细项可以删除。

需要补充：
- 精简框架下，行为标准要承接被合并的细分工作。
- 行为模块少，不等于行为标准少。
- 行为要项名较宽时，行为标准应在单元格内编号拆分。

### 1.4 句式结构过于一致

问题表现：
- 大量句子使用“基于……完成……形成……确保……”结构。
- 可读性稳定，但容易显得机械。

需要补充：
- 可接受的句式变体。
- 哪些场景适合“基于……完成……”。
- 哪些场景适合“围绕……制定……推动……”或“结合……判断……优化……”。

## 2. 现有内置规则

以下是当前 Skill 已经内置的规则，你可以直接修订。

### 2.1 行为标准数量规则

当前规则：

```text
简单、执行型行为要项：每个适用职级 1 条。
中等复杂行为要项：每个适用职级 2 条。
涉及分析、机制、跨部门推进、风险控制的复杂行为要项：每个适用职级 2-3 条。
不为了凑数量强行写满 3 条。
不制造重复或换词式标准。
```


### 2.2 职级定位规则

当前规则：

```text
专员：执行、收集、整理、跟进、维护、初步分析、基础判断、按标准完成。
主管：独立负责、分析判断、计划制定、过程推动、问题处理、跨部门协同。
经理：统筹、规划、资源配置、机制优化、复杂问题解决、结果负责。
总监：策略制定、体系建设、方向判断、组织协同、重大决策、经营结果。
```

本次建议修订方向：

```text
职级定位不能变成固定套词。每个职级应根据行为要项类型选择不同动作词。
经理级不能默认全部以“统筹”开头。
```


### 2.3 行为标准结构规则

当前规则：

```text
每条行为标准应隐含包含：
1. 行为内容
2. 行为动作
3. 行为结果
4. 衡量标准

推荐句式：
基于【输入/依据/场景】，完成【行为动作】，形成【产出/结果】，确保【衡量标准】。
```

本次建议修订方向：
