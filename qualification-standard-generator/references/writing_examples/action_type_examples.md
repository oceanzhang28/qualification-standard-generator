# 行为动作类型示例库 v0.1

本文件提供行为标准写作参考。示例用于学习动作类型和职级递进方式，不应机械套用。

## 分析类

- 适用：数据分析、问题诊断、趋势判断、原因归因。
- 推荐动作词：收集、整理、分析、判断、识别、归因、诊断、评估、输出、提出。
- 不推荐动作词：了解、掌握、具备、熟悉。

| 职级 | 示例方向 |
|---|---|
| 专员 | 基于既定数据口径，完成基础数据收集、清洗和汇总，形成规范的数据表或分析底稿，确保数据准确、完整、按时提交。 |
| 主管 | 基于业务目标和多维数据，独立完成问题分析和原因判断，形成分析结论与改善建议，确保结论逻辑清晰、能够支撑业务动作。 |
| 经理 | 围绕重点经营问题，统筹跨模块数据分析和专题诊断，识别关键影响因素并提出系统性解决方案，确保分析结果推动业务改善。 |
| 总监 | 基于经营战略和业务趋势，判断关键问题方向和分析框架，推动建立分析机制和决策支持体系，确保专业分析能力支撑经营决策。 |

## 计划类

- 适用：工作计划、项目计划、资源安排、节奏规划。
- 推荐动作词：制定、拆解、排期、统筹、配置、明确、形成。

| 职级 | 示例方向 |
|---|---|
| 专员 | 基于明确任务要求，完成工作计划拆解和进度维护，形成个人执行清单，确保任务按节点推进。 |
| 主管 | 基于模块目标，独立制定专项计划和推进节奏，明确任务分工、关键节点和交付要求，确保模块目标按期达成。 |
| 经理 | 围绕业务目标，统筹多个模块计划和资源配置，形成整体实施方案，确保关键任务协同推进并支撑结果达成。 |
| 总监 | 基于业务战略和年度目标，制定专业领域规划和资源投入方向，推动跨组织计划协同，确保规划支撑经营重点。 |

## 执行类

- 适用：标准动作执行、资料整理、流程处理、日常维护。
- 推荐动作词：执行、处理、维护、跟进、提交、反馈、闭环。

| 职级 | 示例方向 |
|---|---|
| 专员 | 基于既定流程和标准，完成日常事项处理和资料维护，确保结果准确、及时、可追溯。 |
| 主管 | 基于模块工作要求，独立推动标准流程执行并处理常规异常，确保过程闭环、交付稳定。 |
| 经理 | 围绕模块运行效率和质量，统筹执行过程管理和关键问题处理，推动流程优化并保障结果达成。 |
| 总监 | 基于专业体系要求，推动关键流程和执行机制建设，确保组织层面的运行效率和质量稳定。 |

## 推进类

- 适用：项目推进、事项闭环、进度跟踪、问题协调。
- 推荐动作词：推动、跟踪、协调、解决、闭环、拉通。

| 职级 | 示例方向 |
|---|---|
| 专员 | 基于明确任务分工，跟进事项进度和问题反馈，推动责任人按要求完成交付。 |
| 主管 | 独立推动专项工作过程，协调相关方解决常规问题，确保关键节点按计划闭环。 |
| 经理 | 统筹复杂专项或跨模块工作，协调资源并解决关键阻塞，确保项目目标和业务结果达成。 |
| 总监 | 推动跨组织重点事项，明确方向、资源和决策机制，确保重大项目对经营目标形成支撑。 |

## 协同类

- 适用：跨部门协作、需求澄清、资源对接。
- 推荐动作词：澄清、对接、协调、拉通、确认、推动。

| 职级 | 示例方向 |
|---|---|
| 专员 | 根据任务要求完成信息对接和资料传递，确保协作信息准确、及时。 |
| 主管 | 独立澄清协作需求并协调相关方推进，确保模块协作事项按要求完成。 |
| 经理 | 统筹跨部门协同机制和重点问题处理，推动资源配置和责任闭环。 |
| 总监 | 建立跨组织协同规则和决策机制，推动关键资源整合，确保专业工作支撑经营目标。 |

## 监控类

- 适用：过程监控、质量检查、指标跟踪、异常识别。
- 推荐动作词：监控、检查、跟踪、识别、预警、反馈。

| 职级 | 示例方向 |
|---|---|
| 专员 | 按既定标准完成过程数据记录和异常反馈，确保问题及时暴露。 |
| 主管 | 独立跟踪模块指标和过程质量，识别常规异常并推动处理闭环。 |
| 经理 | 建立关键指标监控和问题预警机制，统筹处理复杂异常并推动改进。 |
| 总监 | 从业务目标出发设计专业监控体系，推动风险预警和管理决策联动。 |

## 复盘类

- 适用：结果复盘、问题总结、经验沉淀、优化建议。
- 推荐动作词：复盘、总结、提炼、沉淀、优化、输出。

| 职级 | 示例方向 |
|---|---|
| 专员 | 基于执行结果完成基础复盘和问题记录，形成改进建议。 |
| 主管 | 独立组织模块复盘，识别关键问题并形成可执行优化措施。 |
| 经理 | 统筹重点项目或多模块复盘，提炼共性问题和机制优化方向。 |
| 总监 | 推动专业领域复盘机制建设，将经验沉淀为规则、方法或体系。 |

## 机制建设类

- 适用：机制搭建、流程优化、体系完善。
- 推荐动作词：设计、建立、优化、完善、推动、沉淀。

| 职级 | 示例方向 |
|---|---|
| 专员 | 参与规则和流程执行，反馈执行问题并协助更新材料。 |
| 主管 | 基于模块运行问题，独立优化流程或模板，提升工作稳定性。 |
| 经理 | 统筹模块机制建设和流程优化，形成可复制的工作方法。 |
| 总监 | 设计专业体系和组织级机制，推动跨部门落地并持续迭代。 |

## 风险控制类

- 适用：风险识别、预警、控制措施、合规检查。
- 推荐动作词：识别、评估、预警、控制、检查、处置。

| 职级 | 示例方向 |
|---|---|
| 专员 | 按标准执行风险检查和异常反馈，确保风险信息及时上报。 |
| 主管 | 独立识别模块风险并推动常规风险处置，确保过程可控。 |
| 经理 | 统筹复杂风险评估和跨部门处置，建立预防和闭环机制。 |
| 总监 | 从经营和专业体系角度判断重大风险，推动风险治理机制建设。 |

## 标准制定类

- 适用：标准、规范、模板、方法论制定。
- 推荐动作词：制定、规范、定义、沉淀、推广、迭代。

| 职级 | 示例方向 |
|---|---|
| 专员 | 按既定标准完成执行，并反馈标准使用中的问题。 |
| 主管 | 独立制定模块内操作规范或模板，确保团队执行一致。 |
| 经理 | 统筹制定跨模块工作标准和方法，推动复制应用和持续优化。 |
| 总监 | 建立专业领域标准体系和治理机制，推动组织层面统一应用。 |
