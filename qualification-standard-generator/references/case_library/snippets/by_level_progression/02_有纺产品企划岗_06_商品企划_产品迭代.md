# 职级递进片段：产品迭代

- 来源岗位：有纺产品企划岗
- 行为模块：商品企划
- 成熟度：成熟
- 使用提醒：部分职级间虽有工作细项差异，但具体工作内容存在复制情况。

| 职级 | 行为标准 |
|---|---|
| 专员 | 产品开发及迭代产品的资源需求，协助拉通产品迭代方案中一体化职能立项达成，项目时序，以及渠道新产品开发沟通 |
| 主管 | 1、产品售后管理：收集售前/售后中的用户反馈数据、产品退货/换货数据，参与研发/供应链分析产品退货/换货原因评审，参与输出产品改善方案（退货率/售后率下降） |
| 经理 | 1、产品售后管理：收集售前/售后中的用户反馈数据、产品退货/换货数据，参与研发/供应链分析产品退货/换货原因评审，参与输出产品改善方案（退货率/售后率下降）<br> <br> 2、产品变更&升级管理：进行市场需求分析、竞品分析、销售数据分析、用户反馈、售后数据信息，并参与产品变更评审，输出通过评审的产品变更方案 |
| 总监 | 1、产品迭代升级：通过销售表现，产品售后评价，用户体验、市场洞察等产品信息评估产品迭代方案的可行性，并给出相关专业意见指导产品跨越周期实现生意增长 |
