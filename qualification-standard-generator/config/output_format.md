# 输出格式

## 最终输出顺序

1. 拆解逻辑说明
2. Markdown 预览表
3. 校验报告
4. 关键修订说明
5. 人工复核重点
6. TSV 可复制版

## 固定表格字段

```text
序号
行为模块
行为要项
专员行为标准
主管行为标准
经理行为标准
总监行为标准
```

## Markdown 预览表要求

- 使用固定 7 列。
- 所有岗位都保留专员、主管、经理、总监四列。
- 非适用职级或明显不承担该行为要项的职级单元格留空。
- 单元格内多条标准使用 `1、` `2、` `3、` 编号。

## TSV 要求

- TSV 必须放在最终输出最后。
- TSV 使用独立代码块。
- TSV 前后不要夹解释性文字。
- 使用 Tab 分隔列。
- 单元格内部尽量不换行。
- 多条行为标准在同一单元格中用 `1、` `2、` `3、` 编号连接。

TSV 表头固定为：

```text
序号	行为模块	行为要项	专员行为标准	主管行为标准	经理行为标准	总监行为标准
```
