## 字段颗粒度说明

| 字段 | 需要的颗粒度 | 好示例 | 不足示例 |
|---|---|---|---|
| 职类 | 使用职位图谱中的标准职类 | 产品类 | 商品 |
| 职务子类 | 使用职位图谱中的标准子类 | 产品企划子类 | 企划 |
| 所在部门 | 能定位岗位业务归属 | 商品中心 | 公司 |
| 岗位名称 | 尽量使用标准岗位名 | 有纺产品企划岗 | 企划 |
| 适用职级范围 | 明确选择四级中的哪些 | 专员、主管、经理 | 全部看看 |
| 关键任务 | 5-10 条，每条有动作、对象、产出或目的 | 基于平台销售数据和库存情况，完成品类经营分析，识别销售异常并形成调整建议。 | 做数据分析 |
| 主要工作对象 | 说明工作主要作用对象 | 电商平台、消费者、商品品类 | 业务 |
| 主要工作流程 | 4-8 个关键步骤 | 目标承接 -> 数据分析 -> 货品规划 -> 活动推进 -> 复盘优化 | 运营流程 |
| 主要产出物 | 列出可交付成果 | 月度经营分析、活动方案、商品调整建议、复盘报告 | 报告 |

## 主要工作对象与主要产出物的区别

- 主要工作对象：岗位“处理、研究、设计、运营、支持”的对象，回答“围绕什么开展工作”。
- 主要产出物：岗位完成工作后交付的成果，回答“最终产出什么”。

示例：

| 岗位 | 主要工作对象 | 主要产出物 |
|---|---|---|
| 产品设计岗 | 有纺服装类产品、消费者穿着需求、产品企划方向 | 产品设计方案、设计稿、样品评审意见、订货品方案 |
| 电商运营岗 | 电商平台、店铺、商品品类、用户流量 | 店铺运营计划、活动方案、经营分析报告、复盘报告 |
| HRBP岗 | 业务部门、组织、岗位、干部与人才 | 人力策略建议、人才盘点结果、组织诊断报告、专项方案 |

## 字段规则

- 公司任职资格体系只适用于专业类岗位，管理岗位不适用。
- 职族统一为专业族，不要求用户填写。
- 职位类别在 v0.1 中不适用，不要求用户填写。
- 职类、职务子类、岗位名称应优先匹配 `references/position_map.md`。
- 缺少职类、职务子类、所在部门、岗位名称、适用职级范围、至少 5 条关键任务时，不得生成。
- 缺少特殊职级差异、协作方、典型工作场景、关键业务指标时，可以基于内置规则继续生成。

## 关键任务填写提示

关键任务应尽量包含：动作 + 对象/事项 + 产出或目的。

不合格示例：

```text
负责招聘
做数据分析
跟进项目
支持业务
```

合格示例：

```text
根据年度编制和招聘需求，完成岗位需求澄清、渠道发布、候选人筛选与面试推进，保障关键岗位及时补充。
基于月度销售数据、库存结构及渠道反馈，完成品类经营分析，识别销售异常与库存风险，形成商品调整建议。
```

如果用户输入自由描述或任务质量过低，先整理为 5-10 条候选关键任务，请用户确认后再进入生成。
