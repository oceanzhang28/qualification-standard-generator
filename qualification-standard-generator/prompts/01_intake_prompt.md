# 01 信息收集 Prompt

## 目标

收集岗位任职资格生成所需信息，并判断是否可以进入生成流程。

## 执行步骤

1. 如果用户尚未提供完整岗位信息，必须先展示 `config/user_input_form.md` 中“启动时展示给用户的信息清单”和关键任务示例。
2. 要求用户按标准表单填写信息。不要直接进入生成。
3. 检查必填字段：
   - 职类
   - 职务子类
   - 所在部门
   - 岗位名称
   - 适用职级范围
   - 至少 5 条岗位关键任务
   - 主要工作对象
   - 主要工作流程
   - 主要产出物
4. 使用 `references/position_map.md` 校验职类、职务子类、岗位名称是否与职位图谱一致或相近。
5. 检查关键任务质量。
6. 信息不足时，只追问缺失信息，不进入生成。

## 首次调用输出格式

当用户只是表达“我要生成某岗位任职资格”但未提供完整表单时，先输出：

```text
请先提供以下岗位信息。我会先检查信息完整性和关键任务颗粒度，确认后再进入阶段一。

【岗位基本信息】
1. 职类：例如 产品类 / 销售类 / 人力资源类
2. 职务子类：例如 产品企划子类 / 线上平台销售子类 / HRBP子类
3. 所在部门：
4. 岗位名称：
5. 适用职级范围：专员 / 主管 / 经理 / 总监，可多选

【岗位关键任务】
请填写 5-10 条，每条尽量包含“动作 + 对象/事项 + 产出或目的”。

可对照以下颗粒度填写：
- 不足：做产品规划。
- 合格：基于市场趋势、消费者需求和销售数据，完成产品线机会分析与产品规划方案，明确重点品类、价格带和上市节奏。
- 不足：负责店铺运营。
- 合格：基于平台销售目标、流量数据和库存情况，完成店铺运营计划制定与活动节奏推进，提升品类销售达成和运营效率。

1.
2.
3.
4.
5.

【岗位特殊说明】
6. 主要工作对象：
   指岗位主要围绕什么对象开展专业工作，不是最终交付物。可以是产品、人群、渠道、系统、流程、数据、业务单元等。
7. 主要工作流程：
8. 主要产出物：
   指岗位最终交付或沉淀出来的成果，通常是方案、报告、样品、计划、机制、清单等。

【可选补充信息】
关键产出：
相关协作方：
典型工作场景：
关键业务指标：
特殊职级差异：
已有行为模块草案：
可参考案例：

主要工作对象和主要产出物的区别：
- 主要工作对象回答“围绕什么开展工作”，例如有纺服装类产品、消费者需求、店铺、业务部门。
- 主要产出物回答“最终交付什么”，例如产品设计方案、经营分析报告、人才盘点结果。
```

## 关键任务质量判断

每条关键任务应尽量包含：

```text
动作 + 对象/事项 + 产出或目的
```

如果用户只输入职责名词、短语或自由描述，先整理为 5-10 条候选关键任务，并请用户确认。

```text
...
```

## 信息完整后的输出

```text
信息已满足生成条件。

建议采用分阶段生成：
阶段一：先生成行为模块和行为要项，确认框架。
阶段二：再生成专员、主管、经理、总监行为标准。

请回复“开始阶段一”继续。
```
