# 02 行为模块和行为要项生成 Prompt

## 目标

基于已确认的岗位信息和关键任务，生成行为模块和行为要项框架。

## 必读文件

- `rules/qualification_definition.md`
- `rules/module_item_decomposition_rules.md`
- `rules/mece_rules.md`
- `references/case_library/conversion_report.md`
- 必要时读取相近案例的 `full_positions/` 或 `snippets/by_module/`

## 生成要求

- 只生成行为模块和行为要项，不生成行为标准。
- 说明采用流程化、活动化或混合拆解。
- 说明模块边界。
- 输出关键任务覆盖关系。
- 必须包含固定模块“组织贡献”。
- “组织贡献”固定包含 2 个行为要项：平台/流程建设、人员培养。
- 除组织贡献外，岗位专业模块默认 4-6 个。
- 除组织贡献外，每个模块默认 3-5 个行为要项。
- 如果用户反馈“太细”“希望精简”“参考现有初版模式”，应合并相邻模块，输出精简版框架，并说明合并逻辑。

## 输出格式

```text
【拆解逻辑】

【行为模块与行为要项】
1. 行为模块：
   - 行为要项：
   - 行为要项：

【关键任务覆盖关系】
关键任务 1 -> 行为模块 / 行为要项

【MECE 检查】

【需要用户确认的问题】

请确认以上框架是否可以进入阶段二。v0.1 阶段一最多支持 2 轮调整，请第一轮尽量一次性说全修改意见，包括是否需要精简模块、合并要项、调整命名或补充遗漏任务。回复“确认/继续/没问题”后，我再生成四级行为标准。
```

## 禁止

- 不得在阶段一生成专员、主管、经理、总监行为标准。
- 不得删除组织贡献模块。
- 不得将通用能力词作为行为模块。
