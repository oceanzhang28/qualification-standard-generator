# 岗位性质前置门禁：回归清单

使用 `../fixtures/position-type-gate-regressions.md` 中的三个独立场景。场景一、二检查拒绝与阻断，场景三使用其指定的完整专业岗位夹具检查正向路径。每项记录“通过”或“失败”，并附输出位置或简短原因。

| # | 可观察验收条件 | 通过 / 失败 | 记录 |
| --- | --- | --- | --- |
| 1 | HR 表单将岗位性质限制为且仅为“专业类岗位”或“管理岗位”，并把岗位性质作为阻断字段。 |  |  |
| 2 | 明确填写“管理岗位”时，简要说明本 Skill 仅处理专业类岗位并拒绝继续；不开始事实提炼，不输出审核稿或清洁岗位信息包。 |  |  |
| 3 | 岗位性质缺失、含糊或非枚举值时，状态保持“材料待补充”，要求用户明确选择“专业类岗位”或“管理岗位”，不开始事实提炼。 |  |  |
| 4 | 不得从岗位名称中的“经理”“专家”等字样或访谈内容推断岗位性质。 |  |  |
| 5 | 场景三明确填写“专业类岗位”后可进入事实提炼，事实审核稿保留该值；审核期间如被改为管理岗位或含糊值，继续阻断确认与交接。 |  |  |
| 6 | 场景三格式化清洁岗位信息包前再次验证审核稿中的岗位性质为已确认的“专业类岗位”；清洁交接可不复制岗位性质字段。 |  |  |
