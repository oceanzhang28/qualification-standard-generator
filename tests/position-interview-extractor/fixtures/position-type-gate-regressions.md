# 岗位性质前置门禁回归场景

场景一、二分别以对应 HR 岗位基本信息片段开始新的独立会话；除展示的岗位性质外，其余必填字段和完整访谈转写稿均视为已提供。场景三直接复用一份可执行的完整专业岗位夹具。三个场景不得继承彼此的状态。

## 场景一：明确的管理岗位

```text
【岗位基本信息】
职类：人力资源
职务子类：组织管理
所在部门：人力资源部
岗位名称：组织发展经理
岗位性质：管理岗位
```

## 场景二：含糊的岗位性质

```text
【岗位基本信息】
职类：人力资源
职务子类：组织发展
所在部门：人力资源部
岗位名称：组织发展专家
岗位性质：待确认
```

两个岗位名称都不能作为岗位性质的判定依据。

## 场景三：明确的专业类岗位

以 `complete_mixed-level-transcript.md` 的完整内容开始新的独立会话。该夹具已明确填写“岗位性质：专业类岗位”，用于执行进入事实提炼、在审核稿中保留岗位性质，以及格式化前再次验证岗位性质的正向路径。
